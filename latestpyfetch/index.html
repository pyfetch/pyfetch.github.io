PyFetch v1.2

PyPersonalization

PyFetch v1.2 will be bringing features that should've came with v1.0. It's more of personalization.

New profiles feature

The profiles feature is about giving users seperate config files. Like for example, if you create a profile named "Work", it will be saved as ~/.config/pyfetch/profiles/Work.ini. This is useful if you want to use a seperate config file, test a config file, or just mess around with a config file.

Why should I use the feature?

Because profiles are the same as pyfetch.conf, but instead more portable. It's not like you're able to use the config file outside of ~/.config/pyfetch/profiles (That is a future feature), but it's still techinically portable. You can share it like your pyfetch.conf while you can keep using your main config file.
